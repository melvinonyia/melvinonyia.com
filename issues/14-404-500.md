# Styled 404 + 500 with Cmd-K hint

## What to build

Replace the framework default error pages with on-brand 404 and 500 routes. Each renders the standard header and footer, a short message in the site's voice, a link back to the home page, and a visible hint that ⌘K is available. Activating the hint (click or keyboard) opens the command palette so a visitor on a dead URL can recover without leaving the page.

`/404` must be the response for every URL outside the v1 surface — including all old `/blog/*`, `/work/*`, `/links`, `/press`, and `/stack` paths from the previous site.

## Acceptance criteria

- [ ] `/404` renders with header, footer, on-brand message, Home link, and visible ⌘K hint
- [ ] `/500` renders with header, footer, on-brand message, Home link, and visible ⌘K hint
- [ ] Clicking the ⌘K hint opens the command palette
- [ ] Every URL outside the v1 surface returns the 404 page with a 404 status
- [ ] Head snapshot tests for both error routes
- [ ] Manual verification: visiting `/blog/anything`, `/links`, `/press`, `/stack`, `/some-random-path` all return the on-brand 404
